## Список основных экранов (окон)

### 1.1 - Main Dashboard

> Единое рабочее пространство без лишних панелей.

**Top Bar (Верхняя панель):**
- Window Controls: Кастомные кнопки управления окном (Close, Minimize, Maximize).
- Global Search: Поле ввода "Type to ponder..." с иконкой поиска.
- Suggested Tags: Горизонтальный ряд быстрых фильтров (например: PNG, SVG, Text, Python).
- Action Group: Кнопки "Sort" (Сортировка) и "Filter" (Расширенные фильтры).
- Branding: Логотип Splatera (желтый абстрактный глиф, напоминающий закреплялку).

**Content Area (Рабочая область):**
- Masonry Grid, адаптивная сетка, смешивающая карточки изображений и текстовые блоки разной высоты.

## 2 - Элементы интерфейса

**Типы контента:**

- Image: Превью изображения (`object-fit: cover`).

## 4 - Пользовательские сценарии

### Сценарий 1: Импорт смешанного контента (Code + Design)

1. Пользователь выделяет в системе папку проекта, где лежат `.svg` иконки и `.css` стили.
2. Перетаскивает папку в окно Splatera.
3. **Система:**
    * Сканирует файлы.
    * Для `.svg`: рендерит картинку, определяет цвета. Присваивает тег `SVG`.
    * Для `.css`: читает начало файла, подсвечивает синтаксис. Присваивает тег `Code`, `CSS`.
4. Карточки появляются в общей сетке вперемешку, выстраиваясь оптимально по высоте.

### Сценарий 2: Поиск сниппета и копирование

1. Пользователь помнит, что сохранял код сортировки на Python.
2. Вводит в поиск: "sort".
3. Сетка фильтруется. Пользователь видит карточки, в названии или содержании которых есть слово "sort"
4. Наводит курсор -> нажимает кнопку "Copy" (иконка двух квадратов) на оверлее карточки.
5. Код скопирован в буфер обмена.

### Сценарий 3: Визуальный поиск по цвету

1. Пользователю нужен референс в оранжевых тонах.
2. Нажимает "Filter" -> выбирает Оранжевый цвет в color-picker.
3. Сетка показывает только изображения с оранжевой гаммой. Текстовые файлы скрываются (так как у них нет цвета).
4. Пользователь зажимает карточку мышкой и перетаскивает её (Drag-out) прямо в окно Photoshop/Figma.

## 5. Типовые ошибки и сообщения

1. **Слишком большой текстовый файл:**
    * *Условие:* Пользователь кидает лог-файл на более чем 5 Мб.
    * *UI:* Карточка с иконкой файла и надписью "Preview unavailable (File too large)".
2. **Бинарный файл без расширения:**
    * *Условие:* Файл не распознан как изображение или текст.
    * *UI:* Дженерик-карточка с вопросительным знаком. Тег `Unknown`.
3. **Файл удален из системы:**
    * *Условие:* Попытка открыть Quick Look для удаленного файла.
    * *UI:* Карточка становится полупрозрачной (Disabled), поверх иконка "Broken Link". Текст: "File missing".
4. **Пустой поиск:**
    * *Условие:* Фильтр не нашел совпадений.
    * *UI:* Пустой экран с иллюстрацией и текстом "No artifacts found".
