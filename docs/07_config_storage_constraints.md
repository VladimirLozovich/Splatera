## 1 - Конфиг-параметры

Настройки приложения, которые могут быть изменены пользователем или системой:

*   `theme` (string): `dark` (Цветовая тема интерфейса).
*   `thumbnail_size` (int): `400` (Максимальный размер превью по большей стороне в пикселях).
*   `auto_extract_colors` (boolean): `true` (Автоматически вычислять палитру при импорте картинок).
*   `max_text_preview_lines` (int): `20` (Количество строк для предпросмотра текстовых файлов/кода).
*   `io_timeout_seconds` (int): `10` (Таймаут на чтение очень тяжелого файла, чтобы не вешать систему).

## 2 - Где храним локальные файлы

Используем стандартные пути платформы (через API Tauri `appDataDir`), например:

Windows: `C:\Users\%USERNAME%\AppData\Roaming\Splatera`

*   `settings.json` - Пользовательские настройки (тема, пути к внешним редакторам).
*   `database.json` (или `db.sqlite`) - Индексы файлов, теги, извлеченные цвета и пути к оригинальным файлам.
*   `thumbnails/` (директория) - Сгенерированные сжатые копии изображений для быстрой отрисовки сетки.

## 3 - Политика кеша

В нашем случае кеш это сгенерированные превью и вычисленные данные.

*   **Что кешируем:** Файлы `*_thumb.jpg` (превью изображений) и фрагменты текста для кода.
*   **Ключ кеша:** `hash(absolute_file_path + file_modified_date)`. Если файл изменился на диске — кеш инвалидируется.
*   **TTL (Время жизни):** Бессрочно (до ручной очистки пользователем через настройки или пока исходный файл не будет удален/перемещен).

## 4. Ограничения и лимиты (Constraints)
*   **Max Text File Size:** Файлы кода/текста тяжелее `5 MB` не читаются для превью (показывается только иконка), чтобы не переполнять оперативную память фронтенда.
*   **Max Image Process Size:** Изображения тяжелее `50 MB` не обрабатываются для извлечения цвета в автоматическом режиме (только по клику).
*   **Grid Limit:** Одновременный рендеринг в DOM не более `100` карточек (используется виртуализация/lazy-loading для списков из 1000+ элементов).
*   **Платформа:** Работает только в рамках прав текущего пользователя ОС (без прав администратора).

## 5. Политика ошибок
*   **Что показываем пользователю:**
    *   *Файл удален:* Карточка становится неактивной (opacity: 0.5), иконка "Broken Link", текст: "Исходный файл перемещен или удален".
    *   *Ошибка импорта:* Toast-уведомление (всплывашка) "Файл [Имя] не поддерживается или поврежден".
    *   *Нет доступа:* Toast "Нет прав на чтение папки [Путь]".
*   **Что логируем (в `app.log` внутри папки приложения):**
    *   Паники (Panics) бэкенда Rust.
    *   Ошибки I/O (`std::io::Error`) с указанием полных путей файлов.
    *   Ошибки парсинга JSON/БД.
