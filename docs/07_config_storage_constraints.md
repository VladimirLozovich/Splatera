## Конфиг-параметры

> Настройки приложения, которые могут быть изменены пользователем или системой:

| **Параметр**             | **Тип (Rust)** | **Значение по умолчанию**       | **Описание**                                                              |
| ------------------------ | -------------- | ------------------------------- | ------------------------------------------------------------------------- |
| `library_path`           | `String`       | *(задается при первом запуске)* | Путь к папке, где лежит база данных (`db.json`) и сгенерированные превью. |
| `theme_mode`             | `String`       | `"dark"`                        | Цветовая тема интерфейса (`"dark"`, `"light"`, `"system"`).               |
| `thumbnail_size`         | `u32`          | `600`                           | Максимальный размер превью по большей стороне в пикселях.                 |
| `auto_extract_colors`    | `bool`         | `true`                          | Автоматически вычислять палитру при импорте картинок.                     |
| `max_text_preview_lines` | `u32`          | `20`                            | Количество строк для предпросмотра текстовых файлов или кода.             |
| `io_timeout_seconds`     | `u64`          | `10`                            | Таймаут на чтение очень тяжелого файла, чтобы не блокировать систему.     |
| `items_per_page`         | `u32`          | `50`                            | Количество элементов для пагинации/виртуализации сетки.                   |
## Где храним локальные файлы

Используем стандартные пути платформы (через API Tauri `appDataDir`), например:

Для Windows: `C:\Users\%USERNAME%\AppData\Roaming\Splatera`

| settings.json                                                 | database.json                                                         | thumbnails/ (директория)                                              |
| ------------------------------------------------------------- | --------------------------------------------------------------------- | --------------------------------------------------------------------- |
| Пользовательские настройки (тема, пути к внешним редакторам). | Индексы файлов, теги, извлеченные цвета и пути к оригинальным файлам. | Сгенерированные сжатые копии изображений для быстрой отрисовки сетки. |

## Политика кеша

В нашем случае кеш это сгенерированные превью и вычисленные данные.

*   **Что кешируем:** Файлы `*_thumb.jpg` (превью изображений) и фрагменты текста для кода.
*   **Ключ кеша:** `hash(absolute_file_path + file_modified_date)`. Если файл изменился на диске — кеш инвалидируется.
*   **TTL (Время жизни):** Бессрочно (до ручной очистки пользователем через настройки или пока исходный файл не будет удален/перемещен).

## Ограничения и лимиты (Constraints)

| Ограничение            | Описание                                                                                                                                 |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Max Text File Size     | Файлы кода/текста тяжелее `5 MB` не читаются для превью (показывается только иконка), чтобы не переполнять оперативную память фронтенда. |
| Max Image Process Size | Изображения тяжелее `50 MB` не обрабатываются для извлечения цвета в автоматическом режиме (только по клику).                            |
| Grid Limit             | Одновременный рендеринг в DOM не более `100` карточек (используется виртуализация/lazy-loading для списков из 1000+ элементов).          |
| Платформа              | Работает только в рамках прав текущего пользователя ОС (без прав администратора).                                                        |
## Политика ошибок

**Что показываем пользователю:**
- Файл удален: Карточка становится неактивной, иконка "Broken Link", текст-заглушка.
- Ошибка импорта: Toast-уведомление (всплывашка) "Файл X не поддерживается или поврежден".
- Нет доступа: Toast "Нет прав на чтение папки X".

**Что логируем:**
- Паники (Panics) бэкенда Rust.
- Ошибки I/O (`std::io::Error`) с указанием полных путей файлов.
- Ошибки парсинга JSON/БД.
