> Тут описаны обязательные требования и хотелки.
## Обязательные требования (MUST)

| **Требование**              | **Описание**                                                                                                                                             | **Критерий приёмки (Как проверить)**                                                                                                                                                         |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Импорт файлов**           | Пользователь должен иметь возможность перетащить изображение или папку в окно приложения.<br><br>Иначе, пользователь может просто нажать кнопку импорта. | <ol><li>Файл перетащен или файл загружен через системный диалог</li><li>В json-базе появилась запись</li><li>Карточка отображается в сетке. Файл на диске остался на месте.</li></ol>        |
| **Генерация превью**        | При импорте приложение должно создавать сжатую копию изображения в служебной папке.                                                                      | <ol><li>Исходник весит 10Мб</li><li> В папке приложения появился файл превью весом ~200Кб. Сетка не лагает при скролле.</li></ol>                                                            |
| **Извлечение цвета**        | При импорте приложение должно определять доминантные цвета изображения.                                                                                  | <ol><li>В свойствах карточки после импорта виден набор HEX-кодов (палитра) начиная с доминантного</li></ol>                                                                                  |
| **Фильтрация по цвету**     | Пользователь может выбрать цвет в фильтре, и сетка покажет подходящие изображения.                                                                       | <ol><li>Выбран желтый цвет</li><li>В сетке остались только картинки с желтыми оттенками.</li></ol>                                                                                           |
| **Быстрый просмотр**        | Пользователь может открыть изображение на весь экран нажатием на соответствующую кнопку.                                                                 | <ol><li>Нажата кнопка раскрытия карточки</li> <li>Открылось модальное окно с оригиналом изображения.</li></ol>                                                                               |
| **Автодобавление тегов**    | Начальный тег должен появлятся сразу с импортом файла, исходя из расширения файла.                                                                       | <ol><li>Файл импортирован в программу</li><li>Тег присвоился исходя из расширения файла</li><li>Тег сохранился после перезапуска приложения</li><li>Поиск по тегу находит элемент.</li></ol> |
| **Ручное добавление тегов** | Пользователь может присвоить произвольные текстовые теги изображению.                                                                                    | <ol><li>Тег введен в инпут</li><li>Тег сохранился после перезапуска приложения</li><li>Поиск по тегу находит элемент.</li></ol>                                                              |
| **Обработка битых ссылок**  | Если исходный файл удален, приложение должно визуально это показать.                                                                                     | <ol><li>Исходный файл удален через Проводник</li><li>В приложении на карточке появилась иконка "Broken Link".</li></ol>                                                                      |
## Желательные требования (NICE)

1. Создание виртуальных папок внутри приложения (например, "Логотипы", "UI Киты").
2. Переключение между темами и полная кастомизация интерфейса через CSS.
3. Возможность перетащить картинку ИЗ приложения в Photoshop или Figma.
4. Сортировка по дате добавления, по имени, по размеру.
