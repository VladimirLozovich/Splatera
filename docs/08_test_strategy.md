## Уровни тестирования

### Unit-тесты (Логика и утилиты)

1. **Детектор файлов:** Передача строки `file.py` -> возвращает тип `Code`, язык `Python`.
2. **Извлечение цвета:** Передача мок-изображения (красный квадрат) -> алгоритм должен вернуть HEX `#FF0000` в списке доминантных цветов.
3. **Парсер тегов:** Преобразование строки "logo, ui, dark" в массив `["logo", "ui", "dark"]`.

### Integration-тесты (Связка слоев и I/O)

1. **Чтение БД:** Сохранение записи в локальный JSON и успешное чтение ее обратно (проверка сериализации/десериализации).
2. **Обработка битых путей:** Вызов команды `read_file_head` для удаленного файла -> должен вернуться корректный Error Result (а не паника приложения).
3. **Генерация превью:** Проверка, что после вызова команды импорта в папке `thumbnails` физически создается `.jpg` файл.

### E2E / Acceptance-тесты (Сценарии пользователя)

1. **Smoke тест:** Запуск приложения -> отображение пустого экрана "Drop to stash" без ошибок в консоли.
2. **Сценарий импорта:** D&D файла -> появление карточки в UI -> появление верного бейджа (PNG/TXT) на карточке.
3. **Сценарий фильтрации:** Выбор цвета в Filter Panel -> скрытие из UI всех карточек, не содержащих данный цвет.

## 2. Матрица покрытия требований (Traceability Matrix)

| **Описание MUST**           | **Уровень теста** | **Суть проверки**                                                                                                                                                           |
| --------------------------- | ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Импорт файлов**           | E2E               | Проверка загрузки через Drag&Drop и кнопку системного диалога. Валидация появления записи в JSON, рендера карточки в сетке и отсутствия изменений исходного файла на диске. |
| **Генерация превью**        | Integration       | Импорт тяжелого файла (например, 10Мб) -> Проверка успешного создания сжатой копии (~200Кб) в служебной папке приложения.                                                   |
| **Извлечение цвета**        | Unit              | Валидация алгоритма на тестовых картинках: проверка генерации корректного массива HEX-кодов (начиная с доминантного) и вывода в свойства карточки.                          |
| **Фильтрация по цвету**     | E2E               | Выбор цвета в UI-фильтре (например, желтого) -> Проверка скрытия нерелевантных карточек и отображения только изображений с нужными оттенками.                               |
| **Быстрый просмотр**        | E2E               | Клик по кнопке раскрытия на карточке -> Проверка появления модального окна, в котором отрендерен оригинал изображения (а не превью).                                        |
| **Автодобавление тегов**    | Integration       | Импорт файла -> Проверка автоматического парсинга расширения, записи тега в базу, его сохранения после рестарта и успешной работы поиска.                                   |
| **Ручное добавление тегов** | E2E               | Ввод произвольного текста в инпут тегов -> Проверка сохранения связи с изображением после перезапуска приложения и успешной работы поиска.                                  |
| **Обработка битых ссылок**  | E2E               | Удаление исходного файла средствами ОС (вне приложения) -> Проверка корректной обработки ошибки чтения и появления иконки "Broken Link" на карточке в UI.                   |
