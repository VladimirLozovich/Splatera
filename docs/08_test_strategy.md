## 1 - Уровни тестирования

### Unit-тесты (Логика и утилиты)

1.  **Детектор файлов:** Передача строки `file.py` -> возвращает тип `Code`, язык `Python`.
2.  **Извлечение цвета:** Передача мок-изображения (красный квадрат) -> алгоритм должен вернуть HEX `#FF0000` в списке доминантных цветов.
3.  **Парсер тегов:** Преобразование строки "logo, ui, dark" в массив `["logo", "ui", "dark"]`.

### Integration-тесты (Связка слоев и I/O)

1.  **Чтение БД:** Сохранение записи в локальный JSON и успешное чтение ее обратно (проверка сериализации/десериализации).
2.  **Обработка битых путей:** Вызов команды `read_file_head` для удаленного файла -> должен вернуться корректный Error Result (а не паника приложения).
3.  **Генерация превью:** Проверка, что после вызова команды импорта в папке `thumbnails` физически создается `.jpg` файл.

### E2E / Acceptance-тесты (Сценарии пользователя)

1.  **Smoke тест:** Запуск приложения -> отображение пустого экрана "Drop to stash" без ошибок в консоли.
2.  **Сценарий импорта:** D&D файла -> появление карточки в UI -> появление верного бейджа (PNG/TXT) на карточке.
3.  **Сценарий фильтрации:** Выбор цвета в Filter Panel -> скрытие из UI всех карточек, не содержащих данный цвет.
